Transform Orientation

Transform Orientation determines how the gizmo for Move, Rotate, and Scale is aligned in the viewport. It affects the direction of axis constraints and how transformations are applied.

Transform Orientation works together with the Transform Gizmo, Snap, and Axis Constraints for precise modeling workflows.

How to Access

  • In the viewport, at the middle-top toolbar, next to the Snap (magnet) button, there is a Transform Orientation menu.
  • Click the menu to select between the available orientation types.

Orientation Types

  • Global — aligns the gizmo to the world axes (X, Y, Z). Transformations occur along fixed global directions, regardless of object rotation.
  • Local — aligns the gizmo to the selected object’s local axes. Transformations follow the object’s rotation and orientation.

Behavior

  • When the Transform Orientation is set to Global, axis constraints (X, Y, Z) correspond to world directions.
  • When set to Local, axis constraints follow the selected object’s rotation. For example, the local X-axis rotates with the object.
  • The orientation also affects snapping when transformations are constrained to an axis.

Practical Examples

  • Global: Move a rotated object along the world X-axis — the gizmo stays aligned to the world, not the object’s rotation.
  • Local: Move the same object along its local X-axis — the gizmo follows the object’s rotation.

Choosing the correct orientation helps maintain precision in modeling, especially when transforming rotated objects or working with multiple objects.
